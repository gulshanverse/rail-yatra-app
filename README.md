# 🌟 Overview

RailYatra AI is a next-generation Android application designed to make railway travel smarter, faster, and more convenient through the power of Artificial Intelligence.

Instead of being just another train information app, RailYatra serves as an intelligent travel companion that assists passengers before, during, and after their journey.

Whether users need help searching trains, checking PNR status, planning trips, predicting ticket confirmation, locating station facilities, or interacting with an AI assistant, RailYatra delivers everything in one seamless experience.

The project follows modern Android development principles using Jetpack Compose, MVVM Architecture, Material Design 3, Room Database, Retrofit, and AI integration with Gemini.

Our vision is to transform railway travel by combining intelligent automation, beautiful design, and reliable travel assistance into one unified platform.

# 🏛️ System Design

RailYatra follows a modern layered architecture that separates concerns and ensures maintainability, scalability, and testability.

```text
                    Presentation Layer
                            │
                            ▼
                    Jetpack Compose UI
                            │
                            ▼
                      ViewModel (MVVM)
                            │
                            ▼
                     Repository Pattern
                    ┌─────────┴─────────┐
                    ▼                   ▼
            Local Data Source     Remote Data Source
            (Room Database)       (Retrofit APIs)
                    │                   │
                    ▼                   ▼
             Local Persistence     Gemini / Firebase / Railway APIs
```

**Prerequisites:**  [Android Studio](https://developer.android.com/studio)
1. Open Android Studio
2. Select **Open** and choose the directory containing this project
3. Allow Android Studio to fix any incompatibilities as it imports the project.
4. Create a file named `.env` in the project directory and set `GEMINI_API_KEY` in that file to your Gemini API key (see `.env.example` for an example)
5. Remove this line from the app's `build.gradle.kts` file: `signingConfig = signingConfigs.getByName("debugConfig")`
6. Run the app on an emulator or physical device
